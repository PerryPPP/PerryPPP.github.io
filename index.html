数据分析学习笔记day01

最近正在尝试系统地学习数据分析的相关知识>.< 本着督促自己以及方便别的同学入门数据分析的目的，定期在此处更新我的学习笔记~

Linux文件系统介绍以及局域网配置

####Linux的文件系统概述

  • Linux文件系统使用树型目录结构，在整个文件系统如同一棵倒长的树
  • Linux系统中总是将文件系统挂载到树型目录结构中的某个目录节点中使用
  • 对于Linux系统而言，“万物都是文件，万物皆可挂载”
  • 根是所有目录的起始点 /

####根目录下主要的子目录
在此列举，详细资料请参看书本0.0

  • /bin:包含二进制文件，即可执行程序，这些程序是系统必需的文件，一般的命令都存储在这里
  • /sbin:也用于存储二进制文件，只有超级用户root才可以使用
  • /etc:存放配置文件，如passwd,inittab等 在Linux系统中，所有的配置文件都是文本文件，需要更改配置时直接修改文件内容即可
  • /boot:系统引导时使用的文件，系统中非常重要的内核vmlinux就放在该目录下
  • /dev:存放设备文件，用户可以通过这些文件访问外部设备
  • /home:保存除管理员用户外，其他用户的主目录
  • /lib:存放程序运行时所需要的库文件，动态链接库，公有库都存放在该目录下 ps.yum会自动查询下载依赖
  • /temp:存放各种临时文件
  • /mnt:挂载软盘，光盘，u盘文件的挂载点
  • /root：超级用户的个人主目录
  • /usr:该目录的空间比较大，用于安装各种应用程序
  • /proc：是一个虚拟目录，存放当前内存的映像，由内核自动产生
  • /var:存放一些会随时改变的文件

####局域网工作机制和网络地址配置
用下图的简单示例网络来介绍局域网的配置过程

COMPUTER1（网卡） COMPUTER2（网卡） COMPUTER3
| | |
| | |
|————————>路由器<———————|

首先，对局域网中工作的Computer而言，需要我们配置的信息有：

  • IP：对于每台机器来说是唯一的。是由网关决定的（交换机、路由器）前三个字段与局域网的网关一致，最后一个字段不能重复。需要注意的是xxx.xxx.xxx.255是广播地址，个人设备不能使用。ex.若某路由器的ip为192.168.10.0，那么在此局域网中机器可用的ip地址的最后一个字段的范围就是1~254。
    NETMASK：子网掩码，用来查看网段的 255.255.255.0->二进制：11111111.11111111.11111111.00000000
    用个人的ip与子网掩码在二进制形式下进行与操作，可依次取得前三个段 xxx.xxx.xxx.0
    机器在局域网互连需要网段一致。
    GATEWAY：网关ip，网络的总出口
    DNS：域名解析服务器（域名和ip地址的映射）
    HWADDR（数据链路层）
    局域网内主机名配置（内部识别）：配置hosts文件 /etc/hosts文件中加上对应的ip和你自定义的主机名（每一台都要配置），就可以ping通了
    Ps.在局域网中只能手动对每台电脑进行配置
